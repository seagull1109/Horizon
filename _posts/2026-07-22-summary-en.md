## [Potential counterexample to the Jacobian conjecture discovered](https://terrytao.wordpress.com/2026/07/21/a-digestion-of-the-jacobian-conjecture-counterexample/) ⭐️ 9.0/10

On July 19, 2026, Anthropic employee Levent Alpöge presented an explicit counterexample to the Jacobian conjecture in three-dimensional space, discovered using Claude Fable 5, Anthropic's large language model. This counterexample disproves the conjecture for N > 2, while the case for N = 2 remains unsolved. This is significant because the Jacobian conjecture is a major unsolved problem in mathematics, listed as number 16 in Stephen Smale's list of Mathematical Problems for the Next Century. Its resolution would have profound implications for algebraic geometry and related fields, potentially opening new avenues of research and challenging long-held assumptions about polynomial functions. The counterexample involves a polynomial of degree seven, where the Jacobian determinant (which should theoretically be a polynomial of degree up to 18 in three variables) has all non-constant coefficients vanish through massive cancellation. This involves 1329 coefficients, which is significantly larger than the 84 coefficients in the original polynomial.

hackernews · jeremyscanvic · Jul 21, 21:09 · [Discussion](https://news.ycombinator.com/item?id=48998362)

**Background**: The Jacobian conjecture is a mathematical conjecture concerning polynomials in several variables. It states that if a polynomial function from an N-dimensional space to itself has a Jacobian determinant that is a non-zero constant, then the function has a polynomial inverse. The conjecture was first stated for two variables by Ludwig Kraus in 1884 and then restated for integer-coefficient polynomials in N variables in 1939 by Ott-Heinrich Keller. It has been notorious for the large number of published and unpublished false proofs that contained subtle errors.

## [Poolside.ai Launches Laguna S 2.1 AI Model](https://poolside.ai/blog/introducing-laguna-s-2-1) ⭐️ 8.0/10

Poolside.ai has introduced Laguna S 2.1, a 118B parameter Mixture-of-Experts (MoE) model with 8B activated parameters per token that supports a context window of up to 1M tokens. This model is generating significant community interest and technical discussion, with users comparing it to major models like GPT-5.2 and exploring its practical applications and hardware requirements. Laguna S 2.1 is a competitive AI model that fits achievable home hardware requirements, though users are requesting quantization options for lower-end systems, and it's already being integrated into projects like Mozilla's Otari.

hackernews · rexledesma · Jul 21, 17:17 · [Discussion](https://news.ycombinator.com/item?id=48995261)

**Background**: Mixture-of-Experts (MoE) models are a type of neural network architecture that uses multiple "expert" sub-networks, where only a subset of these experts is activated for each input token. This approach allows for larger total parameter counts while maintaining efficient computation. The context window size refers to the maximum number of tokens the model can process at once, with larger windows enabling more comprehensive understanding of longer texts. Recent advances in AI model development have focused on both increasing model size and optimizing performance for various hardware configurations.
